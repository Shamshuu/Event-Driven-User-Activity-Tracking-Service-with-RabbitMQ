Project Purpose

This project implements a distributed Producer-Consumer pattern using a microservices architecture. Its main goal is to demonstrate asynchronous communication between isolated services where one service generates data/events and another processes them.

Functionalities Implemented

producer-service/:
This service acts as the data generator. It exposes an API or runs a scheduled task to create messages or events and pushes them to a message broker (e.g., Kafka, RabbitMQ) or directly to the database.

consumer-service/: 
This service listens for the events generated by the producer. Once it receives a message, it processes the data (e.g., performing calculations, transforming data) and likely persists the results.

Database Initialization (init.sql):
Provides the initial schema and seed data for the application's persistent storage, ensuring the database is instantly ready when the containers start.

Technologies Used and How They Help

Java & Maven (pom.xml): 
The core programming language and build tool used for both services. Maven handles dependency management and builds the applications into executable artifacts (found in the target/ directories).

Docker (Dockerfile):
Containerizes each Java service, ensuring they run in isolated, reproducible environments regardless of the host machine.

Docker Compose (docker-compose.yml):
Orchestrates the multi-container setup. It defines how the producer-service, consumer-service, the database, and likely a message broker interact with each other over a shared virtual network.

SQL (init.sql):
Used to define relational database tables and structures, ensuring the consumer or producer has a reliable place to store and query application state.
Environment Variables (.env.example): Manages configuration settings (like database passwords or broker URLs) outside of the codebase, ensuring secure and flexible deployments.
